```
Result:   https://tikit-api-xyz123.railway.app/api/v1
```

## ğŸ”§ All Other Values (Auto-Configured)

These are **automatically filled** by Vercel - you don't need to do anything:

### NEXT_PUBLIC_API_BASE_URL
- **Auto-filled from:** Your `NEXT_PUBLIC_API_URL`
- **Value:** Same as API_URL but without `/api/v1`
- **Example:** `https://tikit-api.railway.app`
- **You do:** Nothing (automatic)

### NEXT_PUBLIC_APP_NAME
- **Value:** `TIKIT`
- **You do:** Nothing (pre-configured)

### NEXT_PUBLIC_APP_URL
- **Value:** Auto-generated by Vercel
- **Example:** `https://tikit-system-abc123.vercel.app`
- **You do:** Nothing (Vercel creates this)

### Build Environment
- **NEXT_PUBLIC_ENABLE_ANALYTICS:** `false` (pre-configured)
- **NEXT_PUBLIC_ENABLE_NOTIFICATIONS:** `true` (pre-configured)

### Security & Performance
- **Security Headers:** Auto-configured
- **Performance Settings:** Auto-configured
- **Function Timeout:** Auto-configured (30s)

## â“ Common Questions & Troubleshooting

### Q: I don't have a backend deployed yet. What should I do?

**A:** Use the **"Deploy Demo"** button instead!
- It works without a backend
- Perfect for testing the frontend
- Deploy the backend later and update the URL

### Q: Where do I deploy the backend?

**A:** We recommend **Railway** (easiest):
1. Go to https://railway.app
2. Sign up (free)
3. Click "New Project"
4. Connect your GitHub repository
5. Select the `backend` folder
6. Deploy!
7. Copy the URL Railway gives you
8. Add `/api/v1` to the end

**Full backend deployment guide:** See `backend/README.md`

### Q: What if I enter the wrong URL?

**A:** No problem! You can update it:
1. Go to your Vercel dashboard
2. Select your project
3. Go to Settings â†’ Environment Variables
4. Find `NEXT_PUBLIC_API_URL`
5. Click Edit
6. Paste the correct URL
7. Redeploy

### Q: Can I use localhost URL?

**A:** No, localhost URLs don't work in production:
- âŒ `http://localhost:3001/api/v1`
- âŒ `http://127.0.0.1:3001/api/v1`

You **must** deploy your backend to a public platform (Railway, Render, etc.)

### Q: Do I need to add any other URLs or settings?

**A:** Nope! Just the one backend URL. Everything else is automatic.

### Q: What if the deploy button doesn't work?

**A:** Manual deployment steps:
1. Go to https://vercel.com/new
2. Import your repository: `https://github.com/cabdelkhalegh/TIKIT-SYSTEM-`
3. Set **Root Directory:** `frontend`
4. Add environment variable:
   - **Key:** `NEXT_PUBLIC_API_URL`
   - **Value:** Your backend URL with `/api/v1`
5. Click Deploy
